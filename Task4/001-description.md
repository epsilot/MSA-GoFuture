# Сбор и обработка данных для выполнения задач ценообразования

- Так как Kafka является центральной точкой обмена информации в системе, данные из неё можно использовать в потоковом
  режиме для сбора метрик для выполнения ML анализа. Для этих целей можно использовать Kafka Connect.
- Сырые данные выгружаются в хранилище откуда они в дальнейшем могут быть прочитаны модулем обучения моделей
  ценообразования, а также инструментами BI.
- Скомпанованные метрики выгружаются в быстродоступные кеш, где они будут использованы ML моделяем для текущего
  инференса.
- При необходимости рассчёта цены модуль бронирования публикует соответсвующее событие и ожидает ответного события о
  выполненном рассчёте от ML модуля.
